# Content Idea Machine — Changelog

## v1.3.0 (2026-05-15)
**Diagnosed weaknesses:**
1. Content calendar is rigid — same Day 1-5 sequence regardless of whether creator is new or established
2. Top 5 selection criteria is virality-only — doesn't adapt for B2B (pipeline) or commerce (sales) goals
3. No anti-pattern warnings — most common failure mode (topic statements formatted as hooks) goes unaddressed

**Changes:**
- Split content calendar into two sequences: "New/small audience" (lead with tactical value) and "Established audience" (lead with bold perspective), with guidance on which to pick
- Updated top 5 selection to flex by context: audience builders = virality, B2B = pipeline/authority, commerce = sales-adjacent value
- Added "Anti-patterns" section with 3 specific failure modes: topic-as-hook, "How to" clustering, and hooks that need expansion to make sense
- Net change: +20 lines (179 -> 199). Calendar got longer but now serves two distinct use cases.

**Anchors verified:** Output format unchanged (still 5 pillars + 20 ideas + top 5 + calendar). Core identity unchanged. Tone unchanged. All 6 key principles preserved.

**Final state:** 199 lines, up from 155. Growth is +28% — all additions are enforcement mechanisms, edge case handling, and quality safeguards. No prose bloat.

## v1.2.0 (2026-05-15)
**Diagnosed weaknesses:**
1. Pillar definitions are creator-only — B2B brands and commerce creators get awkward "personal story" / "behind-the-scenes" labels that don't fit
2. Adversarial case (zero context) has no ask-back template — first impression quality is model-dependent
3. Multi-platform requests get flattened to single-platform — loses value for IG+TikTok or LinkedIn+Twitter combos

**Changes:**
- Added "Context adaptation" block under pillars: B2B gets relabeled pillars (founder story, industry commentary), commerce creators get 2-3 content-to-sales bridge ideas
- Added ask-back directive for zero-context inputs — energetic, one-message, "help me help you" tone with example phrasing
- Changed multi-platform handling: primary + secondary with cross-posting notes instead of "pick one"
- Net change: +6 lines (173 -> 179). All additions improve edge case handling.

**Anchors verified:** Output format unchanged. Core identity unchanged. Tone unchanged. All 6 key principles preserved.

## v1.1.0 (2026-05-15)
**Diagnosed weaknesses:**
1. Hook patterns too narrow — only 4 types, producing monotone hooks across 20 ideas
2. No self-check before output — ideas drift toward similarity without a quality gate
3. Format recommendations ignore user's platform — suggesting long-form articles to TikTok creators

**Changes:**
- Expanded hook patterns from 4 to 7 (added question, pattern interrupt, list tease) with explicit rotation rule (max 4 uses per pattern)
- Added platform filter to Framework 2 — formats must match the user's stated platform or be explicitly adapted
- Added Step 4 "Quality gate" — internal self-check for uniqueness, hook pattern diversity, and swap test before output
- Net change: +18 lines (155 -> 173). All additions are enforcement mechanisms, not prose.

**Anchors verified:** Output format unchanged. Core identity unchanged. Tone unchanged. All 6 key principles preserved.
